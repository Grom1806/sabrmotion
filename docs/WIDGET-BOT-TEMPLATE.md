# Widget-Bot Template — продай как продукт

Готовый плавающий чат-виджет (тизер + сценарный маркетинг-диалог + живой AI на свободный текст, 3 языка). Этот документ — чек-лист, как за ~15 минут перенастроить виджет под нового клиента и продать.

---

## 1. Что входит в продукт

| Возможность                   | Где живёт                                         |
| ----------------------------- | ------------------------------------------------- |
| Плавающая кнопка + пульс-glow | `components/ChatBot.tsx` (нижняя `motion.button`) |
| Тизер-облачко с ротацией фраз | `teasers` в `ChatBot.tsx`                         |
| Сценарный диалог с кнопками   | `flows` в `ChatBot.tsx`                           |
| Свободный текст → живой AI    | `app/api/chat/route.ts` (Groq, стриминг)          |
| 3 языка RU/EN/DE              | `lib/translations.ts` + `Lang`                    |
| Фирменный цвет / тёмная тема  | CSS-переменные в `app/globals.css`                |

**Зависимости:** Next.js (App Router), React, Framer Motion, `groq-sdk`.
**Внешние требования:** бесплатный `GROQ_API_KEY` (console.groq.com) для AI-ветки. Сценарная часть работает без бэкенда.

---

## 2. Чек-лист настройки под клиента

Пройди сверху вниз — каждый пункт это один блок, который надо заменить.

- [ ] **Telegram / контакт** — `TG_HANDLE` и `TG_LINK` вверху `components/ChatBot.tsx`
- [ ] **Тизер-фразы** — `teasers` (3-6 фраз на язык)
- [ ] **Сценарий диалога** — `flows` (приветствие, ветки услуг, CTA)
- [ ] **Системный промпт AI** — `SYSTEM_PROMPT` в `app/api/chat/route.ts` (услуги, цены, контакт)
- [ ] **Фирменный цвет** — `--accent` и связанные в `app/globals.css`
- [ ] **Имя/статус бота** — заголовок панели в `ChatBot.tsx` (`Sabrmotion`, `headerStatus`)
- [ ] **Языки** — если клиенту нужен 1 язык, оставь один в `Lang`/словарях
- [ ] **`.env.local`** — пропиши `GROQ_API_KEY`

## 3. Шаблоны для заполнения

### 3.1 Контакт (ChatBot.tsx, верх файла)

```ts
const TG_HANDLE = '@{{CLIENT_TELEGRAM}}'
const TG_LINK = 'https://t.me/{{CLIENT_TELEGRAM}}'
```

### 3.2 Тизер-фразы

```ts
const teasers: Record<Lang, string[]> = {
	ru: [
		'{{Зазывающая фраза 1}} 👋',
		'{{Боль клиента в виде вопроса}}',
		'{{Главная услуга одной строкой}}',
		'Оставьте заявку — отвечу быстро ⚡',
	],
	// en / de — те же фразы на других языках
}
```

Правила: коротко, выгода/боль, 1 эмодзи, последняя фраза — призыв.

### 3.3 Сценарий (одна ветка = одна услуга)

```ts
flows.ru = {
	start: {
		bot: [
			'Привет! 👋 Я бот-консультант {{BRAND}}.',
			'{{Питч: что делаем + чем отличаемся}}.',
			'Что вас интересует?',
		],
		replies: [
			{ label: '🅰️ {{Услуга A}}', next: 'svcA' },
			{ label: '🅱️ {{Услуга B}}', next: 'svcB' },
			{ label: '💬 Просто вопрос', next: 'about' },
		],
	},
	svcA: {
		bot: [
			'{{Зацепка про услугу A}}',
			'{{Выгоды + **жирным** ключевые слова/цены}}',
			'{{Условия работы / гарантия}}',
			'Обсудим ваш проект?',
		],
		replies: [
			{ label: '📝 Оставить заявку', next: 'lead' },
			{ label: '✈️ Написать в Telegram', next: 'telegram' },
			{ label: '↩️ Назад', next: 'start' },
		],
	},
	// svcB, about — по тому же шаблону
	lead: {
		bot: [
			'Супер! 🚀',
			`Напишите мне в Telegram **${TG_HANDLE}** — отвечу быстрее всего.`,
			'Жмите кнопку ниже 👇',
		],
		replies: [
			{ label: '✈️ Открыть Telegram', next: 'telegram' },
			{ label: '↩️ В начало', next: 'start' },
		],
	},
}
```

**Спец-значения `next`:** `telegram` открывает ссылку, `start` сбрасывает диалог. Любой другой ключ должен существовать как шаг в `flows`.

### 3.4 Системный промпт AI (app/api/chat/route.ts)

```
You are {{BOT_NAME}} — assistant for {{BRAND}}, {{короткое описание}}.

## Services & Pricing
- {{Услуга A}}: {{цена}}
- {{Услуга B}}: {{цена}}
- Add-ons: {{...}}

## Key facts
- {{сроки}} · {{условия оплаты}} · {{контакт}}

## Behaviour
- Reply in the SAME language as the user
- Keep it short (chat bubble): **bold** + emoji + line breaks, NO ## headings
- Never invent prices/features not listed above
- End with a CTA to {{TG_HANDLE}}
```

### 3.5 Фирменный цвет (app/globals.css)

```css
:root {
  --accent: {{HEX основного цвета}};
  --accent-hover: {{светлее}};
  --accent-border: {{rgba accent 0.28}};
  /* остальные --accent-* пересчитать под новый HEX */
}
```

## 4. Запуск и проверка

```bash
npm install
# в .env.local: GROQ_API_KEY=...
npm run dev      # http://localhost:3000
npm run build    # проверка типов перед сдачей
```

Проверь руками: тизер появляется и ротируется → кнопка открывает/закрывает чат → кнопки ведут по веткам → свободный текст получает AI-ответ → смена языка перезапускает диалог на новом языке.

## 5. Варианты упаковки для продажи

| Тариф        | Что входит                                                                |
| ------------ | ------------------------------------------------------------------------- |
| **Базовый**  | Только сценарный диалог + тизер (без AI, без `GROQ_API_KEY`)              |
| **Стандарт** | Сценарий + живой AI на свободный текст                                    |
| **Премиум**  | + интеграция заявок (Telegram-уведомление / CRM / БД вместо плейсхолдера) |

**Куда расти (следующая продажа):** заявки сейчас уводят в Telegram-ссылку. Подключение реального приёма лида (бот шлёт заявку в Telegram владельца, в Google Sheet или CRM) — это отдельная платная доработка.

## 6. Памятка перед сдачей клиенту

- Заменены `TG_HANDLE` / `TG_LINK` (нет `@sabrmot1on`)
- В промпте нет чужих (Sabrmotion) данных и цен
- `GROQ_API_KEY` — это **серверный** ключ, держать только в `.env.local` (в `.gitignore`), не коммитить
- Имя бота и `headerStatus` под бренд клиента
- Цвет `--accent` под бренд
